static int get_group_id ( Group *  group,
int  i,
int  target,
int *  cpos 
)
static

Get lexicon ID of grouping element target (1 = target, 0 = source) for item i in query result set.

Returns an integer ID to be used for frequency counts in an n-gram hash. This is either a lexicon ID (if the grouping element refers to a p-attribute) or a virtual ID computed from the offset of the annotation string (if the element refers to an s-attribute, relying on the fact that each unique annotation string is stored only once in the s-attribute).

If cpos is not NULL, the corresponding corpus position is written to *cpos.

The following special cases can occur:

  • A: The grouping element is unused (source in a simple group statement)
    • returns ANY_ID = -2 (which pretty-prints as (all) in the grouping output)
    • corpus position is set to ANY_ID
  • B: The anchor point is undefined (possible for target and keyword anchors)
    • returns -1 (which prints as (none) in the grouping output)
    • corpus position is set to ANY_ID
  • C: The anchor point with offset (= effective cpos) is outside the valid corpus range
    • returns -1 (either explicitly for cpos < 0, or implicitly as returned by cl_cpos2id)
    • corpus position is set to effective cpos, or to -1 for all negative values (to avoid confusion with ANY_ID)
  • D: The effective cpos is valid, but not inside a range of the referenced s-attribute
    • returns -1
    • corpus position is set to effective cpos
Parameters
groupPointer to a data structure describing the group operation (and holding results).
iThe row of the query result to be accessed.
targetWhether to return information for the source element (0) or the target element (1).
cposIf not NULL, the effective corpus position is returned in this location.
Returns
An integer ID to be used in frequency counts (see above for special cases)

Detailed Description

This contains accessor functions for the CL global variables which are defined as "extern" in the include-it-everywhere "globals.h" header.

This allows CQP/CWB-utils to access these variables without including the extern declarations; while the CL can access them qua variables.

It also contains the must-always-be-called cl_startup() function and a couple of other utilities.

Safely add an offset to a corpus position.

Return CDA_EPOSORNG if cpos + offset is outside the corpus (clamp = 0), or clamps the return value to the valid range (clamp = 1). Particular care is taken to avoid integer overflow if cpos is close to INT_MAX.

Parameters
cposcorpus position (must be in valid range)
offsetpositive or negative offset from corpus position
corpus_sizecorpus size as returned by cl_max_cpos()
clampboolean: whether to clamp return value (1) or return an error (0)
Returns
cpos + offset (possibly clamped) or CDA_EPOSORNG

void cl_startup ( void  )

Startup function for the CL.

All programs that use CL should call this before doing anything else.

Currently, all it does is make sure that string case-insensitive comparison will work the same everywhere regardless of the locale set by the system or user.

◆ CorpusList

typedef struct cl CorpusList

The CorpusList object records information on a corpus that CQP recognises.

This might be an actual corpus in the registry (->type=SYSTEM), or a subcorpus or query result (->type=SUB).

Note that CorpusList is a bit of a misnomer, although it IS a linked-list entry object, and CQP keeps information on the "corpora" it currently knows about on such a list, in fact this object is often used for passing around information about individual corpora, queries etc. as well.

Remove a corpus from the global list of corpora.

corpuslist
Parameters
clThe corpus to drop. If NULL, nothing happens.
search_fromIf NULL, dropcorpus searches from the start of the global list. Otherwise, it starts at the point indicated – which must be an item known to be BEFORE cl in the global list, or else the global list itself. (This makes looping through the whole corpuslist, and deleting some, more efficient.)
Returns
Pointer to previous item on the linked list (which now directly precedes the next item) or, if the first item was deleted, to the new head of the list. Only of use if it is fed back to "search_from" in a subsequent iteration of a loop through the corpuslist (but with caution: if the first item was deleted, it now points to the following item in the list, which must not be skipped in the next iteration). Returns NULL if the dropped corpus was not found on the part of the list searched or if it was the only corpus in corpuslist (which is now empty). See drop_temp_corpora().

CorpusList* LoadedCorpus ( char *  name,
char *  qualifier,
CorpusType  type 
)

Finds a loaded corpus.

This function tries to find the corpus with name 'name' in the list of currently loaded corpora. In case of subcorpora, qualifier is the mother's name. in case of system corpora, qualifier is the registry. If qualifier is NULL, it is neglected and the first matching corpus is returned. If type is not UNDEF, only a corpus of that type can be returned. No side effects take place.

Parameters
nameThe corpus we are looking for.
qualifierAn extra "bit" of the corpus name (see function description).
typeWhich type of corpus is wanted (may be UNDEF, which finds any except TEMP).
Returns
Pointer to the CorpusList of the corpus that was found, or NULL if not found.

◆ split_subcorpus_name()

char* split_subcorpus_name ( const char *  corpusname,
char *  mother_name 
)

Splits a query result "corpus"-name into qualifier and local name.

This function splits query result name {corpusname} into qualifier (name of mother corpus) and local name; returns pointer to local name part, or NULL if {corpusname} is not syntactically valid.

if mother_name is not NULL, it must point to a buffer of suitable length (CL_MAX_LINE_LENGTH is sufficient) where the qualifier will be stored (empty string for unqualified corpus, and return value == {corpusname} in this case)

Parameters
corpusname"Corpus" name to process. This is not really a corpus but a query result or subcorpus, in the form CORPUS:TheQuery. It won't be modified.
mother_nameBuffer for a copy of the qualifer (first part of name, aka the mother, aka the corpus). When using split_subcorpus_name as a syntax validator for possibly qualified names, this would normally be NULL.
Returns
Pointer into the corpusname buffer, at the point where the second part (the subcorpus/query name) begins. NULL is returned if, at any point in corpusname, there is a character that should not be there.
